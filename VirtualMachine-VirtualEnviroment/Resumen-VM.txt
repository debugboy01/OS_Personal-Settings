Docker:
Software que mantiene abiertas muchos "contenedores" que en realidad son imagenes de sistemas operativos a la vez. cada sistema puede hacer operaciones pero por recomendacion solo debe hacer uno por contenedor, es contenedor, porque su sesion es similar al de un gestionador de veriones. modificas un OS, gestionado por Docker, y se debe guardar la diferencia en esa imagen.

Vagrant:
Software de virtualizacion, multiples maquinas encendidas, y se conectan mediante shell, se basa en la descarga los VDE(imagenes base de un OS) desde un servidor, donde cada OS puede tener ya ciertas caracteristicas ( a cada OS se le llama "box", debido a que en sus VDE ya contiene configuraciones segun tu quieras ).

VirtualMachinesSoftware (VirtualBox, VMWare):
Software de virtualizacion, instala las iso como cualquier OS o pueden ser ejecutadas desde un VDE.

Qemu:
Software de virtualizacion, usada desde consola para ser lanzadas almacenando una imagen con toda la informacion del OS, se optimiza cuando quieres lanzar OS livianos o de poco consumo. (podria ser algunos OS de prueba), si tu OS es muy pesado, se recomienda usar uno de los anteriores (VirtualMachineSoftware).
